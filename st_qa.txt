2. 알고리즘은 무엇인가?

3. 포인터를 사용하는 이유는 무엇인가?

4. Array와 Linked List를 어떤 기준으로 선택할 것인가?
Array는 연속적인 메모리 공간을 지정, 메모리에 연속성이 좋을때 사용
Linked List는 랜덤하게 지정된 메모리를 연결하는데 사용, 메모리 크기 지정이 자주 변경할 때,
메모리 삭제, 삽입이 자유롭게 이루어 질때 사용

5. 데이터를 여러 자료구조에 담는 이유는 무엇인가?
효율성과 데이터 삽입, 삭제, 탐색, 정렬 등의 목적에 따라 최적의 구조를 선택하지 위해서

6. 컴파일러와 인터프리터의 차이는 무엇인가?
컴파일러는 소스를 기계어로 변환을 하여 사용하고 컴파일러를 사용하지 않고 직접 실행 시 오류를
발생하며 오류를 발생하는 이유는 os? 따라 소스를 해석하는 방법이 달라서 컴파일로 변환하영 실행을 
해야한다 대신 컴파일된 소스는 소스확인이 어려워서(2진수를 해석이 가능 하면 가능) 보안에 좋다

인터프리터는 해당 소스를 바로 실행이 가능하며 컴파일(해당하는 언어)을 가상에서 실행을 지원하는 부분이 존재하여
해당 소스를 바로 실행이 가능 오류 사항을 바로바로 수정 실행할 수 있다

1. 변수는 공간인가 개념인가?
   -> 설계 시에는 개념적 특성에, 구현 시에는 공간적 특성에 중점을 둔다.

2. 알고리즘은 무엇인가?
   -> 특정 목적을 위해 한정된 시간 안에 결과를 내는 방법과 과정의 집합.
   
3. 포인터를 사용하는 이유는 무엇인가?
   -> 현대 소프트웨어는 폰노이만 구조에서 시작됨. CPU는 메모리(RAM)을 주소로만 접근함. 즉, 포인터는 컴퓨터 하드웨어를 가장 직관적으로 구현한 개념.
   -> 포인터를 사용함으로써 얻는 이익들: 데이터의 불필요한 복사 방지, 동적 메모리 할당 

4. Array와 Linked List를 어떤 기준으로 선택할 것인가?
   -> 읽기가 주 사용 사례이며 크기가 고정되어 있고 같은 타입을 저장한다면 Array, 요소 추가와 수정 등이 활발하다면 Linked List
   
5. 데이터를 여러 자료구조에 담는 이유는 무엇인가?
   -> 데이터의 사용 사례에 따라 데이터 구조가 다르면 성능에 큰 차이가 발생하기 때문
   
6. 컴파일러와 인터프리터의 차이는 무엇인가?
   -> 컴파일러는 코드-바이너리-실행, 인터프리터는 코드-인터프리터-실행
   -> 컴파일러는 하드웨어에 따라 따로 컴파일, 인터프리터는 하나의 코드로 다양한 하드웨어에서 실행 가능(물론 예외도 있다.)
   -> 컴파일러가 인터프리터보다 빠른 경우가 많다.

7. C와 Python 중 무엇이 더 좋은 언어인가?
   -> 케이스바이케이스
   -> C는 실행 속도, Python은 개발 속도
   -> Python의 라이브러리가 C로 구현되어 있는 경우도 많다.
   -> 개인적으로 Python을 매우 싫어하지만, AI 호환성 때문에 Python을 주로 사용한다.
